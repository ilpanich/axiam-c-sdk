◆ axiam_headers_t

typedef axiam_kv_t axiam_headers_t

Incoming request headers for the guard.

A framework adapter populates this from the real request. The guard reads:

  • "Authorization: Bearer <jwt>" or
  • "Cookie: axiam_access=<jwt>" (session cookie) to obtain the caller's identity token.

enum axiam_guard_status

Guard outcome — the numeric values are the HTTP status a framework adapter should return (§11.2 error mapping).

Enumerator
AXIAM_GUARD_ALLOW 

Authenticated and authorized.

AXIAM_GUARD_UNAUTHENTICATED 

No/invalid session (authentication_failed).

AXIAM_GUARD_DENIED 

Authenticated but denied (authorization_denied).

AXIAM_GUARD_BAD_REQUEST 

Unresolvable resource id (invalid_request).

AXIAM_GUARD_UNAVAILABLE 

Authz endpoint unreachable — fail closed (authz_unavailable).

§11 require_access.

Verifies the caller's token (JWKS), then checks (action, resource_id[, scope]) for that caller as subject. Returns an axiam_guard_status_t. scope may be NULL.

§11 require_auth.

Returns AXIAM_GUARD_ALLOW when the request carries a valid (JWKS-verified) session token, else AXIAM_GUARD_UNAUTHENTICATED / AXIAM_GUARD_UNAVAILABLE.

§11 require_role (local).

Verifies the token and checks that its roles claim contains at least one of the given roles. No server round-trip. Returns AXIAM_GUARD_ALLOW / AXIAM_GUARD_DENIED / AXIAM_GUARD_UNAUTHENTICATED.
